# Отчет по аудиту безопасности: Payment Service

**Аудитор:** Ralesy

**Дата аудита:** 21 апреля 2026 г.  
**Цель аудита:** Оценка безопасности payment-service. Анализ включает код, конфигурации и зависимости на предмет уязвимостей.  
**Методология:** Статический анализ кода, проверка конфигураций (Dockerfile, Pipfile), анализ зависимостей, сканирование с Bandit.

**Примечание:** Этот отчет фокусируется на payment-service как на автономном компоненте. В полной архитектуре проекта (с gateway и другими сервисами) некоторые риски могут быть компенсированы (например, аутентификацией на уровне gateway). Рекомендуется учитывать это при интеграции.

## Основные уязвимости сервиса

| Уровень риска   | Уязвимость                                | Описание                                                                                                     | Последствия                              | Рекомендации                                         |
| --------------- | ----------------------------------------- | ------------------------------------------------------------------------------------------------------------ | ---------------------------------------- | ---------------------------------------------------- |
| **Критический** | Отсутствие проверки подписи вебхука       | В app.py эндпоинт `/webhook/yoomoney/confirm_payment` обрабатывает данные без валидации подписи от YooMoney. | Подделка платежей, изменение данных.     | Реализовать HMAC-проверку по документации YooMoney.  |
| **Высокий**     | Логирование чувствительных данных         | В app.py логируются user_id, amount, metadata.                                                               | Утечка ПДн.                              | Убрать или маскировать логи.                         |
| **Высокий**     | Отсутствие аутентификации                 | Эндпоинты полагаются на Header('User') без проверки.                                                         | Несанкционированный доступ.              | Добавить JWT-валидацию.                              |
| **Средний**     | Широкая CORS-политика                     | В app.py origins `*`.                                                                                        | CSRF-атаки.                              | Ограничить origins.                                  |
| **Средний**     | Нефиксированные версии зависимостей       | В Pipfile версии `*`.                                                                                        | Уязвимые обновления.                     | Закрепить версии, сканировать CVE.                   |
| **Средний**     | Привязка к всем интерфейсам (Bandit B104) | В main.py host = '0.0.0.0 'привязывает сервер ко всем интерфейсам.                                           | Несанкционированный доступ без фаервола. | Изменить на `127.0.0.1` или добавить фаервол/прокси. |
| **Низкий**      | Отсутствие rate limiting                  | Нет защиты от DDoS.                                                                                          | Перегрузка сервиса.                      | Добавить limiter.                                    |
## Компенсации от связанных сервисов

В полной архитектуре проекта риски могут снижаться за счет других компонентов.

- **Gateway (прокси-сервис):** Может проверять JWT для внутренних запросов и добавлять `User` в headers. Это компенсирует отсутствие аутентификации для `/initiate-payment` и `/subscriptions/valid`. Рекомендация: Интегрировать с gateway для защиты.
- **Общая инфраструктура:** Supabase обеспечивает шифрование данных; другие сервисы могут добавить rate limiting или мониторинг.
- **Внешние сервисы:** YooMoney webhook приходит напрямую — компенсация невозможна без изменений в коде.
## Заключение

В изоляции сервис требует доработок для безопасности. В связке с другими сервисами риски снижаются. Исправьте критические уязвимости перед развертыванием.
